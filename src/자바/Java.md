### 클래스의 생성자

-- 생성자는 리턴타입이 없고 클래스와 이름이 동일해야한다   
-- 매개변수를 다르게 해서 여러개의 생성자를 만들 수 있다

```java
public class Car {
    
    String carName;
    int carNumber;
    
    //기본 생성자
    public Car() {}
    
    //carName 을 받아 저장하는 생성자
    public Car(String carName) {
        this.carName = carName;
    }
    
    //매개변수를 다르게 해서 여러개의 생성자를 만들 수 있다
    public Car(String carName, int carNumber) {
        this.carName = carName;
        this.carNumber = carNumber;
    }

    public static void main(String[] args) {
        Car car = new Car(); //new 예약어를 사용하여 객체를 생성한다
    }
}
```


### 변수의 개념

| 종류                   | 개념                           | 선언방법                                       |
|----------------------|------------------------------|--------------------------------------------|
| 클래스 변수(static 변수)    | 클래스가 시작 될 때 생성, 프로그램 종료시 없어짐 | ```static 변수타입 변수명```                      |
| 지역 변수                | 지역변수를 선언한 중괄호내에서 사용가능        | ```변수타입 변수명```                             |
| 객체 변수                | 객체가 시작 될 때 생성                | ```변수타입 변수명```                             |
| 매개 변수                | 메소드가 호출될 때 생성                | ```public void method(변수타입 변수명) { ... }``` |
| final 변수             | 변수를 수정할 수 없음                 | ```final 변수타입 변수명```                       |


### 변수명 및 메소드명 생성 규칙

- 대소문자가 구분되어야한다
- 예약어를 사용해서는 안된다
- 숫자로 시작되서는 안된다
- 특수문자는 `_`, `$` 만 허용한다

### 추상클래스와 인터페이스

![title](./image/추상클래스와%20인터페이스.png)

#### 추상클래스
```java
abstract class 클래스명 {
    abstract 반환타입 메소드이름();
}

class 클래스명 extends 추상클래스명 { ... }
```

#### 인터페이스
```java
접근제어자 interface 인터페이스명 {
    
    static final 타입 상수명 = 값;
    반환타입 메소드이름(매개변수);
    
}

class 클래스명 implements 인터페이스명 { ... } //인터페이스의 정의된 메소드 몸통을 만들어주어야한다
```

### InputStream, OutputStream

### 쓰레드
-- 다중 작업 환경에서 운영 체제가 여러 프로그램을 병행하여 처리할 때 효율적인 처리를 위하여 분할한 처리 단위

#### 쓰레드 메소드
| 반환 타입        | 메소드 이름(매개변수)                | 설명                   | 
|--------------|-----------------------------|----------------------|
| void         | run()                       | 구현해야하는 메소드           |
| long         | getId()                     | 쓰레드의 고유 id를 리턴한다     |
| String       | getName()                   | 쓰레드의 이름을 리턴한다        |
| void         | setName(String name)        | 쓰레드의 이름을 지정한다        |
| void         | join()                      | 수행중인 쓰레드가 중지할 때까지 대기 |
| void         | join(long mills)            | 매개변수에 지정된 시간까지 대기    |
| void         | join(long mills, int nanos) | 더 자세한 시간만큼 대기        |
| void         | interrupt()                 | 수행중인 쓰레드에 중지요청을 한다   |
| boolean      | isDaemon()                  | 쓰레드가 데몬쓰레드인지 확인      |
| void         | setDaemon(boolean on)       | 데몬쓰레드로 만든다           |
| int          | getPriority()               | 쓰레드의 우선순위를 확인한다      |
| void         | setPriority(int n)          | 쓰레드의 우선순위 값을 변경한다    |
| Thread.State | getState()                  | 쓰레드의 상태를 확인한다        |

###### 데몬 쓰레드
-- 부가적인 작업을 수행하는 쓰레드를 선언할 때 주로 사용한다   
-- 데몬쓰레드는 다른 실행중인 일반 쓰레드가 없으면 멈춘다

###### 우선순위
-- 우선순위의 기본값은 5이다

| 상수            | 값 및 설명               |
|---------------|----------------------|
| MAX_PRIORITY  | 가장 높은 우선순위, 값은 10이다  |
| NORM_PRIORITY | 일반 쓰레드의 우선순위, 값은 5이다 |
| MIN_PRIORITY  | 가장 낮은 우선순위, 값은 1이다   |

### 다형성, 다중 상속, 추상화

#### 다형성
-- 하나의 객체가 여러가지의 타입을 가질 수 있는 것을 말한다

#### 다중 상속
-- 클래스는 다중상속을 사용할 수 없다  
-- 인터페이스는 다중상속을 사용할 수 있다  
-- 인터페이스는 기능이 구현이 되어있지 않아 여러개가 상속이 가능하다   
![title](./image/자바의%20다중상속.png)

#### 추상화
-- 공통된 특징을 묶어 하나의 클래스로 정의하는 것  
-- 다중상속을 사용할 수 없다   
-- 일반 메소드도 사용이 가능하다
