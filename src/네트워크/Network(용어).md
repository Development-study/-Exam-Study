# 네트워크(용어 정리)

### 1장

- `네트워크` : 컴퓨터를 두대 이상 연결하여 서로 데이터를 전송할 수 있는 통신망


- `인터넷` : 전 세계의 컴퓨터를 서로 연결하여 정보를 교환할 수 있도록 만든 하나의 거대한 컴퓨터 통신망


- `패킷` : 작게 분할된 데이터 조각, 네트워크에서 전송하는 데이터의 기본 단위


- `비트` : 정보의 최소 단위, 0 또는 1


- `바이트` : 컴퓨터의 정보량 단위, 8비트 = 1바이트


- `랜` : 비교적 가까운 거리에 위치한 장치들을 서로 연결한 네트워크


- `왠` : 랜을 다시 하나로 묶는 거대한 네트워크


- `인터넷 서비스 제공자(ISP)` : 인터넷에 접속하는 수단을 제공하는 주체


- `서버` : 컴퓨터 네트워크에서 다른 컴퓨터에 서비스를 제공하기 위한 컴퓨터 또는 프로그램


- `DMZ` : 외부 네트워크와 내부 네트워크 사이에 위치한 중간 지대(서브넷)


### 2장

- `프로토콜` : 컴퓨터 간에 정보를 주고받을 때 통신 방법에 대한 규칙이나 표준


- `OSI 모델` : 국제표준화기구(ISO)가 1977년에 정의한 국제 통신 표준 규약


- `TCP/IP 모델` : OSI 모델 7계층을 4계층으로 단순화시켜 사용하는 모델


- `캡슐화` : 컴퓨터 통신에서 상위 계층의 프로토콜 정보를 데이터에 추가하여 하위 계층으로 전송하는 기술


- `역캡슐화` : 상위 계층의 통신 프로토콜에서 하위 계층에서 추가한 정보와 데이터를 분리하는 기술


- `헤더` : 저장되거나 전송되는 데이터의 맨 앞에 위치하는 추가적인 정보 데이터


### 3장

- `물리 계층`: 시스템 간의 물리적인 연결을 하고 전기 신호의 변환 및 제어하는 역할을 담당하는 계층


- `전기 신호` : 전압이 일정 패턴으로 변하여 생기는 일련의 흐름으로 전압의 변화가 모여서 만들어진 신호


- `디지털 신호` : 아날로그 신호와 대비되는 신호 형태, 0 또는 1의 2진수에 대응시켜 나타내는 신호


- `랜 카드` : 컴퓨터의 네트워크 연결 및 데이터 전송을 담당


- `케이블` : 전선, 네트워크를 연결하는 케이블은 랜 케이블 또는 랜선이라고 한다


- `리피터` : 감쇠된 전송 신호를 새롭게 재생하여 다시 전달하는 신호 중계 장치


- `허브` : 랜을 구성할 때 가까운 거리에 있는 장비들을 케이블을 사용하여 연결하는 장치


- `노이즈` : 데이터의 왜곡이나 분해로 인해 전송 매체에서 생기는 전자 신호


### 4장

- `데이터링크 계층` : 네트워크 기기 간에 데이터를 전송하고 물리주소를 결정한다


- `이더넷` : 일반적으로 사용되는 랜에서 가장 많이 활용되는 기술 규격


- `충돌` : 전송 장치 두개가 같은 시점에 패킷을 보낼 때 일어나는 데이터 충돌


- `MAC 주소` : 컴퓨터 네트워크에서 각각의 기기를 구분하기 위해 사용하는 주소


- `스위치` : 랜을 구성할 때 사용하는 단말기 간 스위칭 기능이 있는 통신망 중계 장치


- `전이중 통신 방식` : 송신과 수신이 양쪽에서 동시에 이루어지는 양방향 통신


- `ARP` : 네트워크 계층 주소와 데이터링크 계층 주소 사이의 변환을 담당하는 프로토콜


- `ARP 캐시` : 가장 최근에 변환한 IP 대 하드웨어 주소를 보관하고 있는 램의 한 영역


- `ARP 요청` : MAC 주소를 찾아내기 위해 보내는 브로드캐스트 패킷 요청


- `ARP 응답` : ARP 요청에 대한 응답


### 5장

- `네트워크 계층` : 다른 네트워크와 통신하기 위한 경로 설정을 위해 라우팅을 하며 패킷 전송을 담당


- `IP` : 데이터를 보내는 데 사용되는 네트워크 계층 프로토콜


- `IP 주소` : 장치들이 서로를 인식하고 통신하기 위해 사용하는 주소


- `IP 주소 클래스` : IPv4에서 사용하는 주소 그룹(A, B, C 클래스는 네트워크 ID와 호스트 ID로 구성되고, D 클래스는 멀티 캐스트 주소로 사용, E 클래스는 필요에 따라 사용하기 위해 확보)


- `라우터` : 서로 다른 네트워크를 연결해 주는 장치


- `라우팅` : 패킷을 보낼 때 목적지까지 갈 수 있는 경로를 설정해주는 과정


- `라우팅 테이블` : 목적지 주소를 목적지에 도달하기 위한 네트워크 노선으로 변환시킬 목적으로 사용된다


- `서브넷` : 큰 네트워크를 분할해서 만든 작은 네트워크


- `서브넷팅` : 네트워크를 분할하기 위해 IP 주소의 구성을 변경하는 작업


- `서브넷 ID` : IP 주소의 네트워크 부분을 늘리기 위해 서브넷 마스크로 사용되는 비트


- `서브넷 마스크` : IP 주소의 네트워크 부분만 나타나게 하여 같은 네트워크인지 판별하게 하는 마스크


- `멀티 캐스트` : 한 컴퓨터에서 여러 컴퓨터로 패킷을 동시에 전송하는 것


- `브로드캐스트` : IP 네트워크에 있는 모든 컴퓨터로 데이터를 전송하는 방식


### 6장

- `전송 계층` : 데이터가 중복되거나 누락되지 않고 오류 없이 순서에 맞게 전송되도록 관리


- `연결형` : 데이터를 교환하는 동안 계속 연결을 관리하는 프로토콜의 한 형태


- `비연결형` : 연결에 대한 초기화 과정이 없는 통신


- `TCP` : 전송 계층의 프로토콜, 신뢰할 수 있는 데이터 전송을 보장


- `대역폭` : 정해진 시간동안 전송될 수 있는 데이터의 양


- `UDP` : 정보를 주고받을 때 보내는 쪽에서 일방적으로 데이터를 전달하는 통신 프로토콜


- `3-way 핸드셰이크` : TCP 통신에서 사용하는 신뢰성을 제공하기 위한 통신 방식


- `잘 알려진 포트` : 예약이 되어 있는 포트, 1 ~ 1023번 포트


- `브로드캐스트` : 네트워크의 모든 컴퓨터와 장비에 같은 패킷을 일괄 전송하는 방식


- `일련번호` : TCP 에서 데이터를 보낼 때마다 부여받는 고유한 번호


- `포트 번호` : 컴퓨터가 데이터 통신을 할 때 통신하고자 하는 네트워크 서비스나 특정 프로세스를 식별하는 논리 단위


### 7장

- `응용 계층` : 애플리케이션에 대한 서비스를 제공


- `WWW` : 언제 어디서든 필요한 곳에 접근하거나 정보를 공유하고 주고받을 수 있는 멀티미디어 인터넷 서버


- `HTTP` : 클라이언트와 서버 간에 정보를 주고받기 위해 사용되는 네트워크 프로토콜


- `DNS` : 네트워크에서 호스트 이름을 IP 주소로 변환하는 데 사용하는 시스템


- `FTP` : 파일을 전송하기 위한 프로토콜, 20번 포트를 사용


- `SMTP` : 메일을 송신하는 데 사용하는 프로토콜, 25번 포트를 사용


- `POP3` : 메일을 수신하는 데 사용하는 프로토콜, 110번 포트를 사용


- `HTML` : WWW 를 통해 볼 수 있는 문서를 만들 때 사용하는 프로그래밍 언어


- `URL` : 웹 사이트 주소를 지정하기 위해 사용


### 8장

- `OSI 모델` : 국제 표준화기구(ISO)가 1977년에 정의한 국제 통신 표준 규약


- `헤더` : 저장되거나 전송되는 데이터의 맨 앞에 위치하는 추가적인 정보 데이터


- `트레일러` : 데이터의 맨 뒤에 위치하는 정보, 데이터의 무결성을 보장하는데 사용


- `라우팅` : 패킷을 목적지까지 보낼 수 있는 경로를 설정해 주는 과정


- `스위치` : 랜을 구성할 때 단말기 간 스위칭 기능을 가진 통신망 중계 장치


- `라우터` : 서로 다른 네트워크를 연결해 주는 장치


### 9장

- `무선 랜` : 랜 케이블을 사용하지 않고 컴퓨터 통신을 가능하게 하는 네트워크 방식


- `액세스 포인트` : 무선 인터넷 접속을 도와주는 중계 장치


- `IEEE802.11n` : 부선 네트워크에 사용되는 표준 기술


- `SSID` : 무선 랜을 통해 전송되는 패킷의 각 헤더에 붙는 고유 식별자


- `채널` : 무선 통신에서 하나의 토화 신호나 기타 정보가 전송되는 분리된 전송로
